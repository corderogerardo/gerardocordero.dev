# Translating the courses (en → es)

`public/data/{en,es}/*.json` are **build artifacts** — `scripts/build-data.mjs`
regenerates them from the lesson `.js` dirs on every build. Spanish therefore
lives as *translation overlays* in `tools/i18n-work/<course>/` (committed);
the build applies them onto the English course when emitting the es JSON.
Untranslated modules simply stay English.

| go      | 17      | 57      | 369   | ~23,300     | ✅ modules 0–13; the interview-prep tier (14–16) is still English |
| ios     | 21      | 84      | 585   | ~53,500     | ⬜ |
| android | 21      | 87      | 509   | ~47,100     | ⬜ |
| python  | 28      | 126     | 664   | ~54,200     | ⬜ |
| ruby    | 24      | 135     | 856   | ~72,100     | ⬜ |

Code is safe by construction: `starter`/`solution`/`checks`/`mustNot`/`source`
fields are never extracted, so the build always copies them verbatim from
English. Translators must leave ``` fences and `inline code` inside `md`
untranslated (~2% of md words).

## Workflow

```bash
node tools/i18n-extract.mjs ios          # list modules + word counts
node tools/i18n-extract.mjs ios 0        # -> tools/i18n-work/ios/00-<id>.json
# translate every `text` in that file (keep `path`, `src`, code, URLs as-is)
node scripts/build-data.mjs              # regenerates es JSON with overlays
node tools/i18n-check.mjs                # fails if any overlay drifted
```

One module ≈ 1–4k words — a good unit for one cheap-model (sonnet/haiku)
agent pass. After a course lands, run the learn build and click through
`/es/learn/<course>`.

## Drift, and why every unit carries `src`

Overlay paths are positional (`modules[3].lessons[0].steps[4].q`). Insert a
step into a lesson and every unit after it silently re-points one slot early.
The dangerous case is not a crash — it is a path that *still resolves*, so one
step's translation gets written onto a different step's prose and the site
ships wrong Spanish that nobody reads.

So each unit stores `src`: the English string its `text` was translated from.

- `build-data.mjs` applies a unit only when the English at `path` still equals
  `src`. Otherwise it skips the unit, leaving that string in English — the site
  always builds, and it never shows misplaced Spanish.
- `tools/i18n-check.mjs` turns any skipped unit into a non-zero exit. It runs in
  CI (`validate-learn`), so drift is a failed check, not a silent regression.

When a check fails, **re-point the unit's `path`** at the content its
`src`/`text` actually describe, then refresh `src` to the current English.
Never edit `src` to match whatever now sits at `path` — that silences the
check by creating exactly the bug it exists to catch. If the English prose was
genuinely rewritten, re-extract and re-translate that module.
