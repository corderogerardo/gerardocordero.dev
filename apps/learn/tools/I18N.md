# Translating the courses (en → es)

`public/data/{en,es}/*.json` are **build artifacts** — `scripts/build-data.mjs`
regenerates them from the lesson `.js` dirs on every build. Spanish therefore
lives as *translation overlays* in `tools/i18n-work/<course>/` (committed);
the build applies them onto the English course when emitting the es JSON.
Untranslated modules simply stay English.

## Scope (measured)

| course  | modules | lessons | steps | prose words | status |
|---------|--------:|--------:|------:|------------:|--------|
| go      | 14      | 41      | 278   | ~23,300     | ✅ translated |
| ios     | 21      | 84      | 585   | ~53,500     | ⬜ |
| android | 21      | 87      | 509   | ~47,100     | ⬜ |
| python  | 28      | 126     | 664   | ~54,200     | ⬜ |
| ruby    | 24      | 135     | 856   | ~72,100     | ⬜ |

Code is safe by construction: `starter`/`solution`/`checks`/`mustNot`/`source`
fields are never extracted, so the build always copies them verbatim from
English. Translators must leave ``` fences and `inline code` inside `md`
untranslated (~2% of md words).

## Workflow

```bash
node tools/i18n-extract.mjs ios          # list modules + word counts
node tools/i18n-extract.mjs ios 0        # -> tools/i18n-work/ios/00-<id>.json
# translate every `text` in that file (keep `path`, code, URLs as-is)
node scripts/build-data.mjs              # regenerates es JSON with overlays
```

One module ≈ 1–4k words — a good unit for one cheap-model (sonnet/haiku)
agent pass. After a course lands, run the learn build and click through
`/es/learn/<course>`.

⚠️ Overlay paths are index-based (`modules[3].lessons[0]…`). If lesson `.js`
content is reordered or restructured, re-extract and re-translate the
affected modules — `build-data.mjs` fails loudly on any path that no longer
resolves.
